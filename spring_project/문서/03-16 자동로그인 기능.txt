세션 
	- 서버 
	- 세션 ID는 중복되지 않게 할당.

쿠키
	-클라이언트측
	-보안에 취약
	=> 중요한 정보(회원정보 등)은 쿠키에 저장하면 안됨
	

자동 로그인
	=> 사이트에 접속하면(어떤 url이든 상관 없이)
	자동 로그인 인터셉터가 동작
	=> 세션에 회원 정보가 있는지 확인(로그인 확인)
	=> 로그인 되었으면 건너뜀(자동 로그인 필요 없음)
	=> 로그인 안되어있으면 자동 로그인을 시도
		=> 특정 이름의 쿠키가 있는지 확인
		=> 없을시. 자동 로그인을 원하지 않는 것
		=> 있을 시 자동 로그인을 원하는 것 => 자동 로그인을 시도
`		=> 쿠키의 값을 가져옴
		=> DB에 일치하는 회원 정보가 있는지 확인
		=> 있으면 세션에 저장
