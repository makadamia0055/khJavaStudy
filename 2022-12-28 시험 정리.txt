트리거에 대한 설명

참조관계에 있는 두 테이블에서 하나의 테이블에 삽입 삭제, 업데이트 등의 연산으로 테이블 내용이 바뀌었을 때 
데이터 무결성을 지키기 위해서 관련된 테이블도 연쇄적으로 변경이 이루어지게 작업하는 객체가 트리거.

view
하나 이상의 테이블로 이루어진 가상 테이블 
기본 테이블들 중에서 검색 결과를 별도로 다루고자 할 때 말하는 것을 뷰라고 한다. 


외래키 제약조건

cascade 종속된. 따라서 삭제/수정되는 
set null 	참조되는 기본키가 삭제되는 경우에 해당하는 key값을 null로 바꿈(전제조건 해당키의 도메인이 null 허용일 때 가능)
restrict : 참조하고 있는 경우 삭제가 거부됨.
no action : 지연확인이나 mysql에서는 외래키를 즉시 확인한다. 그래서 mysql에서는 restrict와 동일하다.

외래키 제약 조건에 대한 설명


집합연산에서 union과 union all의 차이
합집합이라는 점에서 같으나
union 중복 제거
union all 중복 제거하지 않음. 그래서 속도 더 빠름


dml 데이터베이스를 관리하는 언어로 조회, 추가, 삭제하는 언어,
ddl 데이터 정의어, 데이터베이스의 객체를 생성, 수정  삭제하는 언어
dcl 데이터를 제어하는 언어로 데이터베이스에 접근하거나 데이터베이스 객체에 접근을 허용/차단하는 언어

(컬럼값을 특정 범위로 제한할 때 사용하는 제약조건) 성적 값을 0점에서 100점까지로 제약할때 사용
mysql에서는 지원안하다가 요즘 지원하는듯. 직접 해보니 지원함.


tcl(Transaction Contrul Language) 트랜잭션 제어 명령어, 롤백(가장 최근 커밋으로 롤백), 커밋(트렌젝션을 반영하여 커밋)

트랜잭션 : 거래라는 뜻으로 데이터베이스 명령어 단위 / 데이터베이스의 일관성을 유지하기 위해
같이 이루어져야하는 작업들을 묶은 것으로 데이터 오류 등으로 중간에 작업이 멈추었을 경우 해당 


커밋 : 묶여있는 작업단위를 실제 적용하는 명령어
롤백 : 묶여있는 작업단위를 이전의 작업단위로 돌리는 명령어
